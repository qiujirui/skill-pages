GitHub Pages

1概述

GitHub Pages Skill 通过 GitHub REST API 直接推送 HTML 页面到 GitHub Pages，完全不需要安装 Git。

核心优势：零安装、零配置、永久地址。只需一个 GitHub Token，即可实现页面的一键部署。

2核心痛点

❌ 传统方式
安装 Git → 配置 SSH → clone → add → commit → push → 等 10 分钟
✅ 本 Skill
一条命令 → Python 调 API → 自动上传 → 秒级部署

3支持的命令

🚀 deploy-all

一键部署：验证 → 创建仓库 → 上传所有页面 → 生成首页 → 启用 Pages

python scripts/github_pages.py deploy-all

📤 deploy

上传单个文件到指定路径

python scripts/github_pages.py deploy <本地文件> <远程路径>

📊 status

查看部署状态：认证、仓库、Pages 是否正常

python scripts/github_pages.py status

📋 list / delete

列出或删除远程文件

python scripts/github_pages.py list python scripts/github_pages.py delete <远程路径>

4配置说明

配置项位置说明
GITHUB_USERNAMEconfig.envGitHub 用户名
GITHUB_TOKENconfig.envPersonal Access Token（需 repo 权限）

6踩坑经验

❌ build_type 不能用 workflow
build_type: workflow 需要 .github/workflows/ 下的 Actions 配置文件，没有的话 Pages 永远不会构建（status 一直为 None），访问 404。
✅ 正确做法
用 source: {branch: main, path: /} 从分支直接部署。删除旧 Pages 配置 → 重新创建 → 等 1-3 分钟生效。

Pages 404 访问不到build_type 用了 workflow确保从分支直接部署，运行 enable-pages 重新配置

文件上传 409并发冲突稍后重试
