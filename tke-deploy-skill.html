1概述

TKE Deploy Skill 是 WorkBuddy 与顶新集团数智科技中心联合打造的内部部署平台，面向前端/后端/全栈工程提供一键上云能力。

你只需准备好工程文件，WorkBuddy 会自动完成分析、打包、上传、构建、部署全流程，最终返回一个可直接访问的公网地址。

3标准工程目录结构

纯前端（最常用）

my-project/ ├── project.yaml # 工程配置文件（必需） ├── README.md # 工程说明（可选） └── frontend/ # 前端代码目录 ├── index.html └── ...

前端 + 后端 + SQLite（推荐架构）

my-project/ ├── project.yaml ├── frontend/ │ ├── index.html │ └── ... ├── backend/ │ ├── app.py │ ├── requirements.txt │ └── ... └── database/ └── init.sql
⚠️ 关键提醒：目录名必须严格为 frontend / backend / database，且 project.yaml 必须在 zip 包根目录！

4project.yaml 配置

纯前端示例

name: my-app version: 1.0.0 description: 我的前端应用 components: - name: frontend type: frontend framework: static port: 80 path: ./frontend

前端 + 后端 + SQLite 示例

name: fullstack-app version: 1.0.0 description: 全栈应用 components: - name: frontend type: frontend framework: static port: 80 path: ./frontend - name: backend type: backend framework: flask version: 3.9 port: 5000 path: ./backend - name: database type: database framework: sqlite path: ./database storage: 1Gi

5部署流程

用户请求 ↓ 检查 JWT Token → 未找到 → 提示联系管理员 ↓ 调用 /admin/create-namespace → 确保命名空间就绪 ↓ 分析工程类型（前端/后端/数据库） ↓ 验证工程格式 → 不符合 → 自动调整 ↓ 打包工程为 zip → 上传到网关 ↓ 调用 /services/deploy → 触发云端构建 ↓ Docker build → push 到 TCR → 创建 K8s 资源 ↓ 返回公网访问地址

6实战经验与踩坑记录

❌ 问题：Docker build failed for frontend
使用 PowerShell Compress-Archive 打包后上传，服务端构建失败。
✅ 解决：改用 Python zipfile 打包
PowerShell 打的包格式有差异，必须用 Python zipfile.ZipFile 以 ZIP_DEFLATED 方式打包。
❌ 问题：Windows ACP 模式限制？
误以为 Windows ACP 会阻止 Python 执行 multipart 上传。
✅ 事实：Python urllib 完全可用
ACP 模式不限制 Python 的网络请求，urllib.request 在 ACP 下可以正常完成 upload + deploy。
❌ 问题：部署接口返回 400 / 404
参数字段名写错，或缺少 file_path。
✅ 解决：严格对照 API 字段
上传接口字段：file, service_name 部署接口字段：file_path（上传返回的）, service_name
❌ 问题：工程目录放错位置
HTML 文件直接放在 zip 根目录，没有 frontend/ 子目录。
✅ 解决：严格标准化目录
所有前端文件必须放在 frontend/ 目录下，project.yaml 在根目录。

7网关 API 端点

网关地址：https://aitools-api.dicos.com.cn/api/v1

端点方法说明
/auth/verifyGET验证 Token 有效性
/admin/create-namespacePOST创建用户命名空间
/services/uploadPOST上传代码包（multipart）
/services/deployPOST触发构建和部署
/servicesGET列出所有服务
/services/<name>GET获取服务详情
/services/<name>/logsGET获取服务日志
/services/<name>DELETE删除服务
