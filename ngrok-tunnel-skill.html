ngrok 内网穿透
让本地服务走向公网

将本地 HTTP 服务通过 ngrok 暴露到公网，
让外部用户可以通过公网 URL 访问本地页面

⚠️ 重要更新 v1.1.0

pyngrok 库有严重 bug：调用 ngrok.connect() 会返回成功消息但实际不启动 ngrok 进程！
正确方法：直接调用 ngrok.exe 本人，不要用 pyngrok 库。

Troubleshooting

故障排查

问题现象 可能原因 解决方案
ngrok.connect() 返回成功但外部无法访问 pyngrok 库的 bug，进程未实际启动 直接调用 ngrok.exe，不要用 pyngrok
本地 8099 正常但公网无法访问 ngrok 进程未运行 检查 tasklist 是否有 ngrok.exe
ERR_NGROK_3200 隧道已离线 重新运行启动命令
隧道建立但无法访问 本地服务器未启动 先运行 start-server 启动本地服务
authtoken 错误 token 过期或格式错误 去 ngrok Dashboard 重新获取
Diagnosis

如何判断问题

三步诊断法，快速定位问题

STEP 1
检查本地服务器
curl http://localhost:8099/
STEP 2
检查 ngrok 进程
tasklist | findstr ngrok.exe
STEP 3
检查 ngrok API
curl http://127.0.0.1:4040/api/tunnels
Lessons

踩坑经验

💀

pyngrok ngrok.connect() / 连接成功但外部无法访问

pyngrok 内部调用有 bug，显示成功但实际未启动 ngrok 进程。必须直接调用 ngrok.exe

🔍

tasklist 找不到 ngrok.exe / 4040 端口无响应

确认 ngrok 进程未运行，直接运行 ngrok.exe 即可恢复

🎯

本地正常但外部失败 / 无进程

问题在 ngrok 侧，不是本地服务器。检查 ngrok 是否启动、authtoken 是否有效
